    public void apply(TpProcessRequest transactionRequest, State stateStore) {
                                  //   ^^^^^^^^^^^^^^^^         ^^^^^^^^^^
                                  // - holds command to    - stores info about
                                  //   be executed           the current state
                                  // - contains payload 
                                  //   bytes opaque to
                                  //   validator core

      // STEP 1 [data-in] unpack command data          
      TransactionData transactionData = getUnpackedTransaction/(transactionRequest);
      // STEP 2 [data-in] retrieve data from the state 
      GameData stateData = getStateData(stateStore, transactionData.gameName);
      // STEP 3 [logic]   exec. business logic         
      GameData updatedGameData = playXo(transactionData, stateData);
      // STEP 4 [data-out] save to state               
      storeGameData(transactionData.gameName, updatedGameData, stateStore);
    }

    private TransactionData getUnpackedTransaction/(TpProcessRequest transactionRequest)
        throws InvalidTransactionException {
      String signer = transactionRequest.getHeader().getSignerPublicKey();
      ArrayList payload = decodeData
                        (transactionRequest.getPayload().toStringUtf8());
      
      return new TransactionData(payload.get(0), payload.get(1), payload.get(2), signer);
    }

    private GameData getStateData(String gameName, State stateStore)
        throws InternalError {
      final String address = name2Address(gameName),
                stateEntry = stateStore.get(address);
      if (stateEntry.length() == 0) return new GameData("", "", "", "", "");
      ArrayList data = decodeData(stateEntry, gameName);
      return new GameData(
        data.get(0), data.get(1), data.get(2), data.get(3), data.get(4));
    }


    private void storeGameData(String gameName, GameData gameData, State stateStore) {
      final String     address = name2Address(gameName);
      final ByteString value   = ByteString.copyFromUtf8(encodeData(gameData));
      Collection> addressValues =
          Collections.singletonList(
               new AbstractMap.SimpleEntry(address, value) );
      Collection addresses = stateStore.set(addressValues);
      if (addresses.size() < 1) throw new InternalError("State Error");
    }

    private ArrayList decodeData(String payload) {
      return new ArrayList<>( Arrays.asList(payload.split(",")) )
    }
    
    private String encodeData(GameData gd) {
      return String.format( "%s,%s,%s,%s,%s",
          gd.gameName, gd.board, gd.state, gd.playerOne, gd.playerTwo);
    }

    CONFIGURING TX FAM. LIST 
     - Let's tell the validator/s to accept TXs from IntegerKey&Settings TX families

Sawtooth provides a Settings transaction family that stores on-chain configuration settings, along with a Settings family transaction processor written in Python.

One of the on-chain settings is the list of supported transaction families. In the example below, a JSON array is submitted to the sawset command, which creates and submits a batch of transactions containing the configuration change.

The submitted JSON array tells the validator or validator network to accept transactions of the following types:

    intkey (IntegerKey transaction family)
    sawtooth_settings (Sawtooth transaction family)

To create and submit the batch containing the new setting, enter the following commands.

$ docker exec -it sawtooth-validator-default bash
root@"validator":/# export JSON='[{"family": "intkey", "version": "1.0"}, {"family":"sawtooth_settings", "version":"1.0"}]'
root@"validator":/# sawset proposal create \
  --url http://rest-api:8008 \
  --key /root/.sawtooth/keys/my_key.priv \
  sawtooth.validator.transaction_families=$JSON \
