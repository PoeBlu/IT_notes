Navigating through a page-book

  • double-click or long-press on a cell to zoom into highlighted content
  • Searching for text:
    1. Enter a text and press the "Regex Search" button to highlight cells with matching content.
    2. Browse visually around the highlighted content to see if the context matches with the search.
    3. Press the "ESC" key in your keyboard or the [show all] link in the top of the screen to show again all cells and their relation with matched content (the neighborhood)
    4. The text is interpreted as a regular expresion.
      Advanced users will profit from this feature to create precise queries
      Normal users can ignore this feature or, if curious, learn about its "power" by clicking [here].
  • Filter by topics:
    1. Click on the labels button (left side of the "Regex Search".
    2. Click on the topics you are interested in
    3. By default will show cells matching *ALL* selected topics. Uncheck the "AND-search" will show cells matching *ANY* of the selected topics.
    4. Close to show the result
    5. Press "ESC" key or [show all] link to show all cells and their relation with matched content (the neighborhood)
NOTE: When just a single cell match the search it will automatically zoom-in the content.
This behaviour allows to create external links to you page that automatically opens the desired cell/content using a link similar to: [this example link]

Sharing content

  • The full page book can be shared as a normal web page by sharing its URL (Universal Resource Locator)
  • If you want to put the attention or highlight some particular topic inside the book,add the query parameter "labels=topic1,topic2,..". For example:
    Share the linux administration page highlighting topics related to audit_user:
    ./DevOps/linux_administration_summary.html?labels=audit_user
    Share the same page, highlight other topic (learn_x_in_y_min):
    ./DevOps/linux_administration_summary.html?labels=learn_x_in_y_min
  • The URL query parameter can be used to share a text-search. For example:
    Share the linux administration page highlighting any cell including the word "disk": ./DevOps/linux_administration_summary.html?query=disk
    Or containing the word "disk" but restricting to the topic audit_user: ./DevOps/linux_administration_summary.html?query=disk&labels=audit_user

Editing the content

  • Content is standard html
  • Use GitHub pull request to request occasional changes.
  • Become a member of The Single-Page-Book Project@GitHub to add you own IT_notes page books or publish in your prefered repository
  • Random UUID can be generated (amongst others) by: clicking here

    The GENERATED UUID can be placed "anywhere" and used to create safe links to concrete "cells" that will work with regular expression search.
    This allow to create and share inmutable URLs similar to http://..../map.html?query=GENERATED_UUID pointing to some cell or region of cells inside the map.
